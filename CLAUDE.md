## TAPPr Website

TAPPr is a homebrewing companion platform. This repo is the public-facing website and REST API built with Next.js 15 (App Router) + Supabase.

## Commands

```bash
npm run dev      # Start dev server with Turbopack
npm run build    # Production build
npm run lint     # ESLint
npm start        # Start production server
```

## Architecture

### App Router Structure

- `src/app/` — Pages and API routes (Next.js App Router)
- `src/app/api/` — REST API endpoints (31 route files)
- `src/components/` — React components; `ui/` contains Shadcn/Radix primitives
- `src/lib/` — Core logic
  - `supabase.ts` — Supabase client init
  - `auth.ts` / `api-key.ts` — API key validation helpers
  - `db/` — Service layer: `beer-service.ts`, `recipe-service.ts`, `review-service.ts`
- `src/types/` — TypeScript interfaces for Brew, Recipe, Review, Route
- `src/db/migrations/` — Supabase SQL migrations
- `src/middleware.ts` — Enforces API key auth on all non-public routes
- `docs/` — API docs, security guidelines, feature docs, migration guides

### Authentication

Header-based API key auth via `src/middleware.ts`. Accepted headers: `X-API-Key`, `x-api-key`, `X_API_Key`, `x_api_key`.

Public routes (no auth): `/api/reviews/public/*`, `/api/beers/public/*`, `/api/waitlist`, `/api/docs`, OPTIONS requests.

Environment keys: `TAPPR_API_KEY` (production), `TAPPR_DEV_API_KEY` (development).

### Data Models

- **Brew** — A beer batch: name, style, ABV, IBU, brew date, keg level, `brewUuid` (from companion app), `apiBrewUuid` (generated by this API)
- **Recipe** — Brewing recipe with normalization for deduplication. `recipeId` is deterministically generated from `platform + author + name` via `generateRecipeId()` in recipe-service.
- **Review** — Three tiers: Quick (1-5 rating), Standard (appearance/aroma/taste/mouthfeel), Expert (detailed BJCP-style scoring)

### Database

Raw SQL via `@neondatabase/serverless` tagged template literals (no ORM). Neon client initialised in `src/lib/neon.ts`. All DB access goes through service functions in `src/lib/db/`. No Prisma, no Drizzle.

```ts
// Query pattern
const result = await sql`SELECT * FROM brews WHERE id = ${id}`;
return result[0] || null;  // result is always an array
```

JSONB fields (`quick_review`, `standard_review`, `expert_review`) require `JSON.stringify()` on write; Neon returns them as already-parsed objects on read.

### Styling

Tailwind CSS v4 with custom HSL variables (amber/orange beer theme), dark mode via `.dark` class. Custom fonts: Patua One (`font-patua`) and Montserrat (default sans).

Path alias: `@/*` → `src/*`
